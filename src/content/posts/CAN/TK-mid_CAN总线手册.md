## CAN总线技术简介

### 什么是CAN总线

CAN (Controller Area Network, 控制器局域网) 是一种串行通信总线协议，最初由德国博世公司在1980年代为汽车行业设计，用于汽车内部电子控制单元之间的通信。如今，CAN总线已广泛应用于工业自动化、医疗设备、船舶、航空航天等领域。

### CAN总线的主要特点

1. **多主节点架构**：CAN总线是多主总线，网络上的所有节点地位平等，任何节点都可以在任意时刻主动发送数据
2. **非破坏性仲裁**：当多个节点同时发送数据时，通过优先级仲裁机制确保高优先级消息优先传输，不会发生总线冲突
3. **错误检测和处理**：内置5种错误检测机制，并支持自动重传，可靠性极高
4. **实时性好**：支持优先级控制，确保关键消息低延迟传输
5. **差分信号传输**：使用CAN_H和CAN_L两根信号线差分传输，抗干扰能力强
6. **短距离高速传输**：支持最高1Mbps波特率（40米距离）和最长10公里传输距离（5Kbps）

### CAN总线的版本演进

| 版本 | 说明 | 特点 |
|------|------|------|
| CAN 1.0 | 初始版本 | 仅支持11位标识符（标准帧） |
| CAN 1.1 | 增强版本 | 兼容CAN 1.0，无新标准 |
| CAN 2.0A | 标准帧规范 | 11位标识符，扩展帧禁止 |
| CAN 2.0B | 扩展帧规范 | 支持11位和29位标识符 |
| CAN FD | 灵活数据速率 | 支持更长数据域（最长64字节）和更高数据波特率 |
| CAN XL | 超大数据域 | 支持最长2048字节数据域 |

### CAN总线的应用领域

- **汽车电子**：发动机控制、车身控制、安全系统、娱乐系统
- **工业自动化**：PLC、传感器、执行器、机器人
- **医疗设备**：诊断设备、监护设备、手术设备
- **农业机械**：拖拉机、收割机、灌溉系统
- **船舶航空**：导航系统、控制系统、通讯系统
- **智能交通**：车辆间通信、交通信号控制

## 硬件系统说明

### CAN总线物理层

CAN总线采用差分信号传输，使用两根信号线：
- **CAN_H**：CAN总线高电平信号
- **CAN_L**：CAN总线低电平信号

#### 信号电平定义

| 状态 | CAN_H | CAN_L | 差分电压 |
|------|-------|-------|----------|
| 隐性电平 (1) | 2.5V | 2.5V | 0V |
| 显性电平 (0) | 3.5V | 1.5V | 2V |

#### 总线拓扑结构

CAN总线采用总线型拓扑结构：
```
终端电阻(120Ω) ──┬── 节点1 ──┬── 节点2 ──┬── 节点3 ──┬── 终端电阻(120Ω)
                 │          │          │          │
              CAN_H/CAN_L CAN_H/CAN_L CAN_H/CAN_L CAN_H/CAN_L
```

### 关键硬件组件

#### 1. CAN控制器

CAN控制器负责实现CAN协议的数据链路层功能，主要功能包括：
- 位定时控制
- 仲裁机制
- 错误检测和处理
- 消息接收和发送

常见的CAN控制器芯片：
- **NXP**：SJA1000、TJA1050
- **Microchip**：MCP2515、MCP2518
- **ST**：bxCAN（集成在STM32系列MCU）
- **Infineon**：TLE7250
- **TI**：SN65HVD230

#### 2. CAN收发器

CAN收发器（Transceiver）是CAN控制器与物理总线之间的接口电路，负责将控制器的逻辑电平转换为CAN总线的差分信号。

主要功能：
- 驱动CAN总线信号
- 接收CAN总线信号
- 过流保护
- 热保护
- 电磁兼容性（EMC）保护

常见的CAN收发器芯片：
- **NXP TJA1040/1041/1042/1043/1044**：高速CAN收发器，支持CAN FD
- **NXP TJA1050**：高速CAN收发器
- **TI SN65HVD230/231/232**：3.3V供电高速CAN收发器
- **Microchip MCP2551/2561**：高速CAN收发器
- **Infineon TLE7250-2G/3G/4G**：高速CAN收发器

#### 3. 终端电阻

终端电阻是CAN总线必不可少的组件，通常使用120Ω电阻，作用是：
- 防止信号反射
- 匹配总线阻抗
- 确保隐性电平稳定

终端电阻必须安装在总线的两端（最远端节点），不能安装在中间节点。

### CAN总线布线规范

#### 线缆选择

推荐使用带屏蔽的双绞线：
- **特性阻抗**：120Ω
- **线缆规格**：AWG 22-24
- **屏蔽层**：编织屏蔽或铝箔屏蔽，屏蔽层接地

#### 布线要求

1. **避免分支**：尽量采用直线总线结构，避免T型分支
2. **节点间距**：节点与总线的连接支线应尽量短（一般不超过0.3米）
3. **远离干扰源**：远离电源线、电机线、高频信号线等干扰源
4. **屏蔽接地**：线缆屏蔽层单点接地（通常在主控制器端）
5. **终端电阻**：总线两端必须安装120Ω终端电阻

#### 传输距离与波特率关系

| 波特率 | 最大传输距离 |
|--------|--------------|
| 1Mbps | 40米 |
| 500Kbps | 100米 |
| 250Kbps | 250米 |
| 125Kbps | 500米 |
| 50Kbps | 1000米 |
| 20Kbps | 2500米 |
| 10Kbps | 5000米 |
| 5Kbps | 10000米 |

## 软件协议说明

### CAN帧结构详解

#### 标准帧（11位标识符）

```
┌──────┬──────────────────┬─────┬──────┬──────┬─────────┬─────┐
│ SOF  │ ID(11位)         │ RTR │ IDE  │ r0   │ DLC(4位)│ Data│
│ 1bit │ 11bit            │ 1bit│ 1bit │ 1bit │ 4bit    │0-8B │
└──────┴──────────────────┴─────┴──────┴──────┴─────────┴─────┘
                              ↓
┌───────────────────────────────────────────────────────────┐
│ CRC(16位) │ CRC分隔符 │ ACK位 │ ACK分隔符 │ EOF(7位) │ IFS(7位) │
│ 16bit     │ 1bit      │ 2bit  │ 1bit      │ 7bit     │ 7bit    │
└───────────────────────────────────────────────────────────┘
```

#### 扩展帧（29位标识符）

```
┌──────┬──────────────────┬─────┬──────┬──────────────┬─────┐
│ SOF  │ ID(11位)         │ SRR │ IDE  │ ID(18位)     │ RTR │
│ 1bit │ 11bit            │ 1bit│ 1bit │ 18bit        │ 1bit│
└──────┴──────────────────┴─────┴──────┴──────────────┴─────┘
                              ↓
┌───────────────────────────────────────────────────────────┐
│ r1  │ r0  │ DLC(4位)│ Data  │ CRC(16位) │ ... │ EOF │
│ 1bit│ 1bit│ 4bit    │0-8B   │ 16bit     │     │7bit │
└───────────────────────────────────────────────────────────┘
```

#### 字段说明

| 字段 | 名称 | 长度 | 说明 |
|------|------|------|------|
| SOF | 帧起始 | 1bit | 显性电平，标志一帧的开始 |
| ID | 标识符 | 11/29bit | 帧标识符，用于仲裁和识别报文 |
| RTR | 远程请求位 | 1bit | 0=数据帧，1=远程帧 |
| IDE | 标识符扩展位 | 1bit | 0=标准帧，1=扩展帧 |
| DLC | 数据长度码 | 4bit | 数据域字节数（0-8） |
| Data | 数据域 | 0-8B | 实际数据 |
| CRC | 循环冗余校验 | 16bit | 错误检测 |
| ACK | 应答位 | 2bit | 接收节点应答 |
| EOF | 帧结束 | 7bit | 标志一帧结束 |
| IFS | 帧间间隔 | 7bit | 帧之间的间隔 |

### CAN报文类型

| 类型 | RTR | IDE | 说明 |
|------|-----|-----|------|
| 数据帧 | 0 | 0/1 | 发送数据 |
| 远程帧 | 1 | 0/1 | 请求数据 |
| 错误帧 | - | - | 通知总线错误 |
| 过载帧 | - | - | 请求延迟下一个数据帧 |

### CAN错误检测机制

CAN总线内置5种错误检测机制：

1. **位错误（Bit Error）**：节点发送的位与检测到的位不一致
2. **填充错误（Stuff Error）**：违反位填充规则（连续6个相同位）
3. **CRC错误（CRC Error）**：CRC校验失败
4. **格式错误（Form Error）**：帧格式不正确
5. **应答错误（ACK Error）**：发送节点未收到应答

### 错误处理机制

CAN节点根据错误状态分为三种状态：

| 状态 | 错误计数范围 | 行为 |
|------|--------------|------|
| **错误主动** | TEC &lt; 128 且 REC &lt; 128 | 正常工作，主动发送错误帧 |
| **错误被动** | 128 ≤ TEC &lt; 256 或 128 ≤ REC &lt; 256 | 只能被动接收，不能主动发送错误帧 |
| **总线关闭** | TEC ≥ 256 | 与总线断开连接 |

其中：
- **TEC** (Transmit Error Counter)：发送错误计数器
- **REC** (Receive Error Counter)：接收错误计数器

### 位填充机制

为了确保总线有足够的跳变沿，CAN总线使用位填充：
- 发送数据时，每连续5个相同位后，自动插入一个反相位
- 接收数据时，自动移除填充位

示例：
```
原始数据：00000
发送数据：000001  （插入反相位）
接收数据：00000    （移除填充位）
```

### CAN标识符分配

CAN标识符（ID）决定了报文的优先级，标识符数值越小，优先级越高。

#### ID分配原则

1. **高优先级报文**（紧急控制、安全相关）使用低ID
2. **周期性报文**按照周期长短分配ID，短周期报文优先
3. **事件触发报文**根据紧急程度分配ID
4. **避免ID冲突**每个报文ID唯一

#### ID分配示例

```
0x000-0x07F：最高优先级，紧急控制
0x080-0x0FF：高优先级，实时反馈
0x100-0x7FF：中优先级，普通数据
0x800-0x1FFFFFFF：低优先级，配置、诊断
```

## TK-mid概述

TK-mid中型履带线控底盘采用CAN2.0B标准进行通信，通过CAN接口可以实现对底盘的运动控制、状态监控等功能。

### 主要功能
- **运动控制**：支持运动学控制和自由控制两种控制方式
- **实时状态反馈**：车速、角速度、轮速、电池状态等
- **I/O控制**：安全停车解锁等功能
- **心跳保护机制**：Alive Counter和Checksum双重保护

### 系统架构
```
┌─────────────────┐         CAN总线 (500Kbps)         ┌─────────────────┐
│   上位机/控制器 │◄──────────────────────────────►│  TK-mid底盘    │
│                 │  0x18C4D1D0 (控制)            │                 │
│  - 运动控制     │  0x18C4D2D0 (自由控制)        │  - 运动执行器  │
│  - 状态监控     │  0x18C4D7D0 (I/O控制)       │  - 传感器     │
│                 │  0x18C4D1EF (运动反馈)         │  - BMS        │
│                 │  0x18C4D7EF/D8EF (轮速反馈)   │  - 遥控器     │
│                 │  0x18C4DAEF (I/O反馈)        │                 │
│                 │  0x18C4E1EF/E2EF (电池反馈)     │                 │

## 控制帧

### 运动学控制指令

**报文信息**
- 报文名称：ctrl_cmd
- ID：0x18C4D1D0
- 类型：Cycle
- 周期：10ms
- 长度：8Byte

**信号定义**
| 信号 | 起始字节 | 起始位 | 长度 | 数据类型 | 精度 | 单位 | 描述 |
|------|----------|--------|------|----------|------|------|------|
| 目标档位 | 0 | 0 | 4 | Unsigned | 1 | - | 00:disable, 01:驻车档, 02:空档, 03:运动学控制档 |
| 目标速度 | 0 | 4 | 16 | Signed | 0.001 | m/s | 0.001m/s/bit |
| 目标角速度 | 2 | 20 | 16 | Signed | 0.01 | °/s | 0.01°/s/bit |
| Alive Counter | 6 | 52 | 4 | Unsigned | 1 | - | 心跳信号，每帧加1 |
| Checksum | 7 | 56 | 8 | Unsigned | 1 | - | 前7字节异或校验 |

#### 数据帧结构详解
```
Byte 0: [档位(4位)][速度高8位(4位)]
Byte 1: [速度低8位(8位)]
Byte 2: [角速度低8位(8位)]
Byte 3: [角速度高8位(8位)]
Byte 4: [保留(8位)]
Byte 5: [保留(8位)]
Byte 6: [Alive Counter(4位)][保留(4位)]
Byte 7: [Checksum(8位)]
```

#### 示例1：切换到运动学控制档

**需求**：将档位切换到运动学控制档，速度0m/s，角速度0°/s

**计算过程**：
```
1. 档位 = 0x03
2. 目标速度 = 0 m/s
   speed_raw = 0 / 0.001 = 0 = 0x0000
3. 目标角速度 = 0 °/s
   angular_raw = 0 / 0.01 = 0 = 0x0000
4. Alive Counter = 0x01
5. Checksum = Byte0 XOR Byte1 XOR Byte2 XOR Byte3 XOR Byte4 XOR Byte5 XOR Byte6

## 使用说明

### 校验和计算

**公式**：
```
Checksum = Byte0 XOR Byte1 XOR Byte2 XOR Byte3 XOR Byte4 XOR Byte5 XOR Byte6
```

**示例计算**：
```
数据: Byte0=0x03, Byte1=0x00, Byte2=0x00, Byte3=0x00, Byte4=0x00, Byte5=0x00, Byte6=0x10
Checksum = 0x03 ^ 0x00 ^ 0x00 ^ 0x00 ^ 0x00 ^ 0x00 ^ 0x10 = 0x13
```

**验证方法**：
1. 计算前7字节的异或值
2. 与接收到的Byte7比较
3. 如果相等，数据有效
4. 如果不等，数据无效，丢弃

### Alive Counter使用

**特点**：
- 占据52至55四个位（字节6的低4位）
- 每发一帧，数值加1
- 达到最大值后从0开始重新计数（0→1→2→...→15→0）
- 用于检测是否丢包和掉线状况

**连续发送示例**：
```
帧1: ... ... ... ... ... ... 00 xx (Alive=0)
帧2: ... ... ... ... ... ... 10 yy (Alive=1)
帧3: ... ... ... ... ... ... 20 zz (Alive=2)
帧4: ... ... ... ... ... ... 30 aa (Alive=3)
...
帧16: ... ... ... ... ... ... F0 bb (Alive=15)
帧17: ... ... ... ... ... ... 00 cc (Alive=0，循环)
```

**接收端检测**：
- 记录上次接收到的Alive值
- 如果新值 = (上次值 + 1) mod 16 → 正常
- 如果不连续 → 可能丢包
- 如果长时间不变 → 可能掉线

### 测试注意事项

1. **发送过程中注意Alive Counter需要连续变化循环发送
2. **测试过程中遥控器切换到指令控制模式或关闭遥控器
3. **使用电脑连接CAN卡测试时，请架起车辆，测试稳定后再放下
4. **落地测试时最好开启遥控器，方便随时切换到遥控模式

### 档位说明

| 档位值 | 描述 | 使用场景 |
|--------|------|----------|
| 0x00 | disable | 禁用控制，底盘不响应 |
| 0x01 | 驻车档 | 紧急停止，电机锁定 |
| 0x02 | 空档 | 自由转动，无动力 |
| 0x03 | 运动学控制档 | 通过速度+角速度控制 |
| 0x04 | 自由控制档 | 通过左右轮独立控制 |

**档位切换流程**：
```
disable → 空档 → 运动学/自由控制 → 空档 → disable
          ↑                               ↓
          └─────── 驻车档 ←─────────────┘
```

### 控制方式选择

- **运动学控制**：通过目标速度和目标角速度控制
  - 优点：符合车辆运动学
  - 适用场景：导航、避障、路径跟踪

- **自由控制**：通过左右轮独立目标速度控制
  - 优点：更灵活，可实现特殊动作
  - 适用场景：原地旋转、特殊操控

- **注意**：两种方式仅且只可以使用一种方式进行控制

### 场景1：直线巡逻

**需求**：底盘以0.5m/s直线前进，持续10秒

**控制流程**：

```
1. 发送档位切换到运动学控制档
   ID: 0x18C4D1D0
   D[0]: 0x03, D[1]: 0x00, D[2]: 0x00, D[3]: 0x00, D[4]: 0x00, D[5]: 0x00, D[6]: 0x00, D[7]: 0x03

2. 持续发送速度指令（10ms周期）
   ID: 0x18C4D1D0
   D[0]: 0x13, D[1]: 0xF4, D[2]: 0x00, D[3]: 0x00, D[4]: 0x00, D[5]: 0x00, D[6]: xx, D[7]: yy

3. 10秒后停止
   ID: 0x18C4D1D0
   D[0]: 0x03, D[1]: 0x00, D[2]: 0x00, D[3]: 0x00, D[4]: 0x00, D[5]: 0x00, D[6]: xx, D[7]: yy
```

**Python代码示例**：
```python
import time
from tkmid_can_example import TKmidCAN

can = TKmidCAN()

# 切换档位
for i in range(10):
    can_id, data = can.build_ctrl_cmd(3, 0, 0)
    # 发送CAN帧...
    time.sleep(0.01)

# 前进10秒
start_time = time.time()
while time.time() - start_time < 10:
    can_id, data = can.build_ctrl_cmd(3, 0.5, 0)
    # 发送CAN帧...
    time.sleep(0.01)

# 停止
for i in range(10):
    can_id, data = can.build_ctrl_cmd(3, 0, 0)
    # 发送CAN帧...
    time.sleep(0.01)
```

### 场景4：状态监控

**监控流程**：
```
1. 订阅运动反馈 (0x18C4D1EF) → 10ms周期
2. 订阅轮速反馈 (0x18C4D7EF, 0x18C4D8EF) → 10ms周期
3. 订阅I/O反馈 (0x18C4DAEF) → 50ms周期
4. 订阅电池反馈 (0x18C4E1EF, 0x18C4E2EF) → 100ms周期
```

**Python监控代码**：
```python
def monitor_status():
    # 解析运动反馈
    ctrl_data = receive_can(0x18C4D1EF)
    ctrl_info = can.parse_ctrl_fb(ctrl_data)
    print(f"档位: {ctrl_info['gear']}, 速度: {ctrl_info['speed']:.2f}m/s")

    # 解析电池反馈
    bms_data = receive_can(0x18C4E1EF)
    bms_info = can.parse_bms_fb(bms_data)
    print(f"电压: {bms_info['voltage']:.2f}V, 电流: {bms_info['current']:.2f}A")

    bms_flag_data = receive_can(0x18C4E2EF)
    bms_flag_info = can.parse_bms_flag_fb(bms_flag_data)
    print(f"SOC: {bms_flag_info['soc']}%")
```

### 常见问题

**Q: 如何判断遥控器是否在控制？**
A: 查看io_fb报文的遥控器控制状态反馈位，1表示遥控器控制，0表示指令控制。

**Q: Alive Counter如何重置？**
A: Alive Counter自动循环，0-15循环，不需要手动重置。

**Q: Checksum计算错误怎么办？**
A: 丢弃该帧数据，等待下一帧。

**Q: 最大速度是多少？**
A: 根据硬件参数，最大速度约4km/h（约1.1m/s）。

**Q: 如何紧急停止？**
A: 1. 按下急停开关 2. 发送驻车档指令 3. 使用遥控器紧急停止
